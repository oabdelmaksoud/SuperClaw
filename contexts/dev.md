# Development Context (OpenClaw)

Mode: Active development
Focus: Implementing features safely and quickly

## Behavior
- Implement in small, verifiable steps.
- Prefer concrete changes over abstract discussion.
- Run relevant checks after edits.
- Keep changes scoped and reversible.

## Priorities
1. Correctness
2. Safety
3. Maintainability
4. Speed

## Tools to favor
- `read`, `edit`, `write` for code/doc changes
- `exec` for local verification/build/test
- `grep/find` via shell for quick discovery

## Output style
- Brief change summary
- Validation evidence
- Remaining risks/next step
