# SKILL_QUALITY_CONTRACT

Required sections in bridge skills:
1. Purpose
2. Trigger Conditions
3. When NOT to Use
4. Workflow
5. Output Format
6. Guardrails

Description requirements:
- Must state what the skill does.
- Must state when it should be used.
- Should include disambiguation hints where overlap exists.

Style:
- Keep `SKILL.md` concise and execution-focused.
- Move long details/examples into `references/`.
